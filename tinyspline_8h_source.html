772 /******************************************************************************
773 * *
774 * :: Transformation functions *
775 * *
776 * TinySpline is a library focusing on transformations. That is, most *
777 * functions are used to transform splines by modifying their state, e.g., *
778 * their number of control points, their degree, and so on. Accordingly, each *
779 * transformation functions specifies an input and output parameter (along *
780 * with the other parameters required to calculate the actual transformation). *
781 * By passing a different pointer to the output parameter, the transformation *
782 * result is calculated and stored without changing the state of the input. *
783 * This is in particular useful when dealing with errors as the original state *
784 * will never be modified. For instance, let's have a look at the following *
785 * code snippet: *
786 * *
787 * tsBSpline in = ... // an arbitrary spline *
788 * tsBSpline out; // result of transformation *
789 * *
790 * // Subdivide 'in' into sequence of bezier curves and store the result *
791 * // in 'out'. Does not change 'in' in any way. *
792 * tsError err = ts_bspline_to_beziers(&in, &out); *
793 * if (err != TS_SUCCESS) { *
794 * // fortunately, 'in' has not been changed *
795 * } *
796 * *
797 * Even if 'ts_bspline_to_beziers' fails, the state of 'in' has not been *
798 * changed allowing you to handle the error properly. *
799 * *
800 * Unless stated otherwise, the order of the parameters for transformation *
801 * functions is: *
802 * *
803 * function(input, [additional_input], output, [additional_output]) *
804 * *
805 * 'additional_input' are parameters required to calculate the actual *
806 * transformation. 'additional_output' are parameters storing further result. *
807 * *
808 * Note: None of TinySpline's transformation functions frees the memory of the *
809 * output parameter. Thus, when using the same output parameter multiple *
810 * times, make sure to free memory before each call. Otherwise, you will *
811 * have a bad time with memory leaks: *
812 * *
813 * tsBSpline in = ... // an arbitrary spline *
814 * tsBSpline out; // result of transformations *
815 * *
816 * ts_bspline_to_beziers(&in, &out); // first transformation *
817 * ... // some code *
818 * ts_bspline_free(&out); // avoid memory leak. *
819 * ts_bspline_buckle(&in, &out); // next transformation *
820 * *
821 * If you want to modify your input directly without having a separate output, *
822 * pass it as input and output at once: *
823 * *
824 * tsBSpline s = ... // an arbitrary spline *
825 * tsReal *knots = ... // a knot vector *
826 * *
827 * ts_bspline_set_knots(&s, knots, &s); // copy 'knots' into 's' *
828 * *
829 * Note: If a transformation function fails *and* input != output, all fields *
830 * of the output parameter are set to 0/NULL. If input == output, your *
831 * input may have an invalid state in case of errors. *
832 * *
833 ******************************************************************************/
